This project follows "constructive frustration" pedagogy:

  1. Study the architecture - Understand the VM design patterns
  2. Identify the challenges - Find intentional implementation gaps
  3. Debug and fix - Learn by solving real problems
  4. Extend and improve - Apply understanding to create better versions
